Git for the rockstar developper
GIT est le gestionnaire de version le plus populaire, utilisé dans de nombreux projets open-source comme AngularJS, NodeJS, Spring Framework, Bootstrap, Gradle...
Il a été créé par Linus Torvalds (auteur du noyau Linux). Contrairement à SVN, sa particularité est d’être décentralisé.

Decentralisé

Un dépôt local GIT contient une copie complète du repository distant dans le dossier caché .git.​

Un dépôt local SVN ne contient que les méta datas des fichiers, le reste est stocké sur le serveur distant.​

Avec GIT les données sont stockées dans le repo local puis un push permet de les envoyer sur le serveur distant. L’avantage est de pouvoir travailler en local indépendamment du serveur, puis envoyer les modifications seulement lorsque c’est nécessaire.

Branches

Pour chaque nouvelles features une branche est créée puis fusionnée à la branche principale, à la différence de SVN où une seule branche principale est utilisée.​
Chaque fonctionnalité est isolée dans une branche spécifique. On s’assure simplement que la branche principale reste stable après l’intégration des différentes fonctionnalités.​
Git basics
Identifier l'auteur de chaque modification du repository 
							
									git config --global user.name "Joani COUMA"
									git config --global user.email joani.couma@cgi.com
Créer un repository 
							
								git init
Clôner un repository distant 
							
								git clone https://github.com/RoanixS2k12/git-presentation.git

Remotes

Pour collaborer avec notre équipe nous avons besoin d’un référentiel commun sur lequel partager nos ressources. Un remote. 
							
								# Ajouter un remote
								git remote add remoteName remoteAdress
								# Lister les remote
								git remote -v
								# Supprimer un remote
								git remote rm remoteName

Repository status

A chaque modification ajout / suppression le repository local possède un état propre 
							
								git status

Pull

Avant de commencer la modification d'un quelconque fichier il est nécessaire d'être à jour. 
							
								git pull remoteName branchName
								git pull origin master
								git pull # Current branch
Il y a d'autre manière de se mettre à jour.

Indexation 1

Chaque changement (modification, ajout, suppression) d'un fichier doit être indexé par Git. 
							
								git add fileName
								git add .
Les fichiers sont alors à l'état staged

Indexation 2

Il est possible de desindexer un fichier 
							
								git reset HEAD fileName

Commits 1

Une fois les modification staged, il est possible de les commit. Le repository local est alors modifié. 
							
								git commit 
								git commit -m 'feat: Initial Commit'
								git commit -am 'feat: Initial Commit'

Commits 2

A chaque commits est associé un idenfiant unique (hash, ou sha1). Il est visible lors de la création d'un commit et dans l'historique. 
							
								git log 
								git log --pretty=oneline --abbrev-commit

Push

Une fois que l'on est satisfait des différents commits locaux il est alors temps de les pousser sur le repo distant... 
							
								git push remoteName branchName
								git push origin master

Bonus: Stash

Lorsque l'on souhaite "mettre de côté" un travail en cours, inachevé 
							
								git stash 	# Mets de côté le travail en cours
								git stash pop 	# Récupère le travail dernièrement remisé 
								git stash list 	# List tous les "états sauvegardés"

Bonus: Reset

Lorsque nous sommes dans un etat bien trop instable... 
							
								git reset HEAD origin/branchName
								git reset --hard origin/branch # Revient à l'état de la branch remote

Bonus: Cherry-pick

Récupère les modifications présente sur un commit exterieur à la branch actuelle, et applique ces modifications 
							
								git cherry-pick #sha1
Advanced GIT
"Je suis en train d’implémenter une nouvelle fonctionnalité sur la branche master (qui est la branche par défaut sous GIT). Mon n+1 me demande de corriger un bug urgent sur un précédent commit. Malheureusement, je suis obligé de terminer mes modifications avant de corriger ce bug… "

Branches 1

Dans ce cas là, la solution aurai été de créer une nouvelle branche afin de pouvoir mettre de côté l'ensemble des commits correspondant à la feature. 
							
								# New branch
								git checkout -b myFeature
								# Retourner sur master
								git checkout master

Branches 2

Chaque projet decidera du nombre de branches / de la manière de gérer ses feature. En général, master correspond aux versions stables de l'application, develop est la version sur laquelle l'équipe travaille. Chaque feature peut être une branche à part.

Branches 3

La commande checkout permet également de se déplacer vers une commit (sha1) voir un tag. 
							
								git checkout 68bdcf9 # Premier commit de cette presentation ..

Branches 4

Pour obtenir la liste des différentes branches du repository local / distant et les différences 
							
								git branch
								git branch -vv
								git branch -av

Branches 5

Pour supprimer une branche du repository local, du remote ou les 2. 
							
								git branch -d localBranch # --delete
								git branch -D remoteBranch # --delete --force
Ainsi que du remote 
							
								git push remote_name --delete branch_name

Merge

On peut merger des modifications d'une autre branche sur la branche courrante. 
							
								git merge master
Aura pour conséquence de rammener toutes les modifications présente sur master sur la branche courante.

Conflits 1

La plupart du temps, lors d'un merge GIT sait reconnaitre quels sont les fichiers modifiés et permet un auto-merging des fichiers. Dans le cas contraire, l'opération est stoppée le temps que les conflits soit resolus.

Conflits 2

Les fichiers présentant un conflit que Git ne sait pas résoudre automatiquement, sont modifiés par Git et il est impératif de résoudre l'intégralité des conflits avant de pouvoir continuer. 
							
								<<<<<<< HEAD:index.html
								// code modifié sur la branche locale
								=======
								// code modifié sur la branche qui est fusionné
								>>>>>>> remoteBranch:index.html

Conflits 3

Attention aux différents outils réalisant des merges conflicts automatiquement. (IntelliJ...)
